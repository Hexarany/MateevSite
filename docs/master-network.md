# Петля «выпускник → мастер»: сеть мастеров под брендом Mateev

Стратегия: выпускник школы (Anatomia) становится bookable-специалистом на сайте
(MateevSite). Замыкаем круг: обучил → сертифицировал → выдал в запись → берёшь
комиссию → расширяешь сеть, не упираясь в свои две руки.

## Роли
- **super-admin (Денис)** — видит всю сеть: все записи, финансы, P&L, склад, всех
  мастеров и клиентов. Это текущая админка (PIN/сессия).
- **specialist (мастер)** — свой **урезанный кабинет**: только своё расписание,
  свои записи/клиенты, свой профиль, своя выручка. НЕ видит чужих данных, финансов
  студии, склада. Реализуется как роль поверх существующей модели (фильтр по
  `specialistId`).

## Модель данных (specialist)
Уже есть: `id, name, role, experience, bio, specialties, daySchedules, photo`.
Добавлено (Фаза 3a): `certified: bool` (бейдж «Mateev-certified»), `location: string`
(город — для мультигорода). План:
- `ownerType: "studio" | "network"` — сам Денис или партнёр сети.
- `commissionPercent: number` — комиссия сети с его сеансов (напр. 15–20).
- `authPhone` / связь с аккаунтом мастера для входа в кабинет.
- (мультигород, позже) `timezone`, `currency` — пока не трогаем (один TZ/валюта).

## Кабинет мастера (Фаза 3b)
Отдельный вход (не PIN супер-админа). Варианты авторизации:
- переиспользовать паттерн client-portal (вход по телефону → токен), но выдавать
  **master-scope** токен, если телефон принадлежит специалисту сети; либо
- отдельный master-логин.
Кабинет = урезанная версия админ-панели: разделы «Моё расписание», «Мои записи»,
«Мой профиль», «Моя выручка», отфильтрованные по `specialistId`. Ставит часы,
выходные/отпуск (переиспользуем уже готовую кнопку закрытия), видит свои записи.

## Комиссия (Фаза 3c)
Рекомендуемая модель: **% с сеанса**, деньги собирает мастер у себя, **сверка раз
в месяц** через счёт от SRL. Система уже пишет каждую запись с ценой и статусом —
отчёт «по мастеру за месяц: N сеансов, сумма, комиссия X%» = P&L, отфильтрованный
на одного мастера. Мягкий старт: первые 1–2 месяца 0%.

## Локация / мультигород
Твой первый кейс — тот же часовой пояс и валюта (другой город Молдовы), поэтому
нужна только `location`. Мультистрана (timezone/currency/выплаты через границу) —
отдельная фаза, включается позже без переделки кабинета.

## Маркетинг (кто что рекламирует)
Спрос — локальный (каждый в своём городе), бренд — общий (Денис). Рычаги роста
для сети: локальные SEO-страницы городов, личные страницы-ссылки мастеров с
записью, отзывы по каждому мастеру, каталог «найди мастера рядом», co-op фонд на
гео-рекламу. Не финансировать чужой локальный спрос из своего кармана.

## Юридическая обвязка (SRL + электронная подпись)
Сеть = франшиза/партнёрство, а не найм. SRL подписывает с мастером договор
(процент, использование бренда, стандарты, «независимый партнёр»), выставляет
счёт на комиссию, может держать товарный знак «Mateev» и лицензировать его.
**Электронная подпись** (получена 01.07.2026) закрывает практическую часть:
подписание договоров с мастерами, регистрация/документы SRL, официальные счета,
e-gov. Конкретику по налогам/форме мастера (PFA/ÎI/patentă/SRL) — с contabil/jurist.

## Фазы
- **3a (в работе):** `location` + бейдж «Mateev-certified» у специалистов (модель,
  админка, публичная карточка). Полезно даже для студии Дениса в одиночку.
- **3b:** кабинет мастера (роль + вход + урезанная панель по `specialistId`).
- **3c:** отчёт по комиссии на мастера (реюз P&L).
- **3d:** публичный каталог/маршрутизация по городам + личные страницы мастеров.
- **later:** мультистрана (timezone/currency/выплаты).

Гейт: полноценная петля стартует, когда первый ученик скажет «да». 3a безопасна и
не зависит от этого.
