# White-label SaaS — стратегия и roadmap

> Компас, а не ТЗ. Цель: превратить платформу, построенную для Mateev Spa Studio, в продукт по подписке для других массажистов и студий. Обновлять по мере валидации.

---

## 1. Что мы продаём

Не «сайт». Мы продаём **готовую операционную систему массажной студии под ключ**:

- Сайт + онлайн-запись (с защитой от двойных броней, окнами, закрытиями студии).
- Личный кабинет клиента (портал, PWA, напоминания).
- **AI-ресепшн 24/7** (сайт + Telegram) — ловит записи ночью.
- CRM: карточки клиентов, история, голосовые заметки → AI-карта, блокнот, долги.
- Дашборд + аналитика (LTV, удержание, топ-услуги) + P&L.
- Маркетинг: генератор постов, «Пост для Google», AI-ответы на отзывы, рассылки, реф-программа, лист ожидания.
- **Школа/сертификация**: дипломы с QR-верификацией, стена выпускников, приватные методички, сеть мастеров (петля) с комиссией.
- AI-помощник владельца + «Диагнозы и снимки».

Строить это с нуля обычному массажисту — нереально. **Мы это уже построили.** В этом весь смысл продукта.

## 2. Кому (ICP — идеальный клиент)

По приоритету:
1. **Соло-массажисты и малые студии** (1–5 мастеров) — им нужно «всё в одном», нет бюджета на разработку.
2. **Выпускники твоей школы** — тёплая аудитория, уже в твоей сети (петля → SaaS): выучился → получил свою мини-платформу от тебя. Гениальный канал: школа кормит SaaS.
3. Косметологи, мануальные терапевты, small wellness (смежные ниши, та же механика).

Гео: старт — Молдова/Румыния (двуязычность RU/RO уже есть — сильное преимущество), далее СНГ/ЕС.

## 3. Почему это может стать больше самой студии

- Студия масштабируется твоим временем (руки, часы). **SaaS масштабируется копированием** — 100 студий по €30/мес = €3000/мес пассивно, без лишних рук.
- Ты уже несёшь затраты на разработку «для себя» — продукт это амортизирует.
- Петля + школа + SaaS = маховик: выпускники становятся клиентами SaaS, платят подписку и комиссию, рекламируют бренд.

## 4. Главный технический барьер: мультиарендность

Сейчас система **одноарендная**: одни данные (JSON-файлы в `data/`), один `.env`, один домен, один набор брендинга. Чтобы продавать — нужна изоляция по студиям («тенантам»):

| Что | Сейчас | Нужно для SaaS |
|---|---|---|
| Данные | общий `data/` | данные на тенант (папка/БД по `tenantId`) |
| Бренд/услуги/контент | один `site.json` | свой на тенант |
| Домен | один | поддомен `studio.платформа.com` или свой домен |
| Секреты (Telegram, AI, почта) | один `.env` | per-tenant конфиг |
| Админ-доступ | один PIN | аккаунты/роли на тенант |
| Биллинг | нет | подписки, триал, отключение неоплативших |
| Онбординг | вручную | самостоятельная регистрация студии |

Это **недели-месяцы**, и требует решений: файлы-на-тенант vs реальная БД (Postgres); маршрутизация по поддоменам в nginx; платёжный провайдер.

## 5. Правило анти-перерасхода: сначала валидация, потом код

**Не строить мультиарендность, пока не подтверждён спрос.** Порядок:

### Этап 0 — Валидация (1–2 недели, БЕЗ кода)
- Поговорить с 5–10 массажистами/студиями (начни с выпускников и знакомых по цеху).
- Вопрос: заплатили бы €X/мес за «сайт + запись + AI + кабинет под ключ»? Что критично, за что готовы платить?
- Определить цену (гипотеза: €19–49/мес по тарифам) и 2–3 killer-фичи, ради которых берут.
- **Критерий перехода дальше:** ≥3 студии сказали «да, плачу» (лучше — предоплата/лист ожидания).

### Этап 1 — Ручной пилот (2–4 недели, минимум кода)
- Развернуть **1 студию на поддомене** вручную (`studioX.<домен>`): копия приложения, свой `data/`, свой `.env`, nginx-блок, TLS через certbot.
- Тот же VPS (экономия). Ручной онбординг (ты заводишь данные за клиента).
- Цель: проверить, что продукт реально работает у другого бизнеса и что он готов платить. Собрать обратную связь.

### Этап 2 — Продуктизация (когда пилот платит и доволен)
- Ввести `tenantId` и изоляцию данных (начать можно с папки-на-тенант, БД — когда упрёшься).
- Провижининг по поддомену (мидлварь резолвит тенант по хосту).
- Self-serve онбординг + биллинг (Stripe/Paddle; для MD/RO — уточнить локальный эквайринг).
- Панель супер-админа (управление тенантами, подписками).

### Этап 3 — Масштаб
- Свои домены тенантов, email-домены, отдельный сервер под тяжёлых, CDN.
- Маркетплейс шаблонов/услуг, реферальная программа для студий (студии приводят студии).

## 6. Хитрость: строить фичи «tenant-ready» уже сейчас

Каждую новую фичу, которую делаем «для себя», проектировать так, чтобы она не мешала будущей мультиарендности:
- Не хардкодить бренд/тексты там, где можно тянуть из `site.json`.
- Изоляция чтения/записи через слой доступа к данным (чтобы позже подставить `tenantId`).
- Настройки — в конфиг-файлы, не в код.

Так фичи из этапов «для себя» становятся частью продаваемого продукта, а не переписываются потом.

## 7. Экономика (грубая гипотеза, проверить на валидации)

- Цена: €19 (соло) / €39 (студия) / €79 (студия + школа/петля) в месяц.
- Затраты на тенанта: хостинг копеечный (один VPS тянет много), основное — AI-токены (перевыставлять или лимитировать), поддержка.
- Точка интереса: ~20–30 платящих студий = стабильный доход уровня самой студии, при почти нулевых доп. затратах времени.

## 8. Риски и как снижать

- **«Построил, а не покупают»** → валидация Этапа 0 перед кодом. Не писать мультиарендность на веру.
- **Поддержка съедает время** → максимум self-serve, база знаний, шаблоны.
- **AI-затраты** → лимиты по тарифу, свой ключ у крупных, кэширование.
- **Расфокус со студии** → SaaS запускать поэтапно, не бросая основной доход; школа как мягкий канал.
- **Юр./данные** → per-tenant изоляция, договор оферты, политика данных (особенно медданные клиентов).

## 9. Ближайший конкретный шаг

**Этап 0.** Составить список из 8–10 массажистов/студий (выпускники + знакомые), провести 20-минутные разговоры по скрипту из §5. Решение о коде — только после ≥3 «плачу».

Когда дойдём — соберу: скрипт интервью, лендинг-заглушку «ранний доступ» для сбора заявок, и прайс-гипотезу.
